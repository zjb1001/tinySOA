# SOME/IP tinySOA 测试计划更新总结

## 更新概览

本次更新对 `someip-bus-test-plan.md` 进行了全面扩展，新增了 **90+ 个测试用例**，将覆盖范围从原有的事件总线 Pub/Sub 模式扩展到完整的 SOME/IP 协议栈，特别是RPC方法调用和拦截器链部分。

## 主要增强点

### 1. **RPC Method 测试全覆盖** (32 个测试)
- **单元测试** (TC-M-001~015): 
  - 方法注册和元数据管理
  - 同步RPC调用（Request/Response）
  - 异步调用（Fire-and-Forget）
  - 参数序列化/反序列化
  - 超时处理
  - 错误处理
  - 并发调用
  - 大型负载处理
  - 方法参数验证

- **系统集成测试** (ST-M-001~010):
  - 端到端RPC调用
  - 多服务RPC链（A→B→C）
  - 大型负载处理
  - 并发压力测试
  - 超时和重试行为
  - 故障场景
  - 负载均衡跨实例
  - 有状态方法的一致性
  - 单向调用语义
  - 版本协商

- **安全测试** (SEC-M-001~003):
  - SQL注入和命令注入防护
  - 反序列化炸弹（Deserialization Bombs）
  - RPC请求洪泛（DoS）

### 2. **拦截器链体系** (27 个测试)
- **执行流程测试** (TC-I-001~008):
  - 拦截器执行顺序验证
  - Before/After/OnError阶段
  - Short-Circuit（短路）行为
  - 上下文传播
  - 异常隔离

- **内置拦截器测试** (TC-B-001~009):
  - **Metrics拦截器**: 计数器、直方图、百分位数
  - **Logging拦截器**: 结构化日志输出
  - **Tracing拦截器**: OpenTelemetry Span创建
  - **Auth拦截器**: 令牌验证、403/401处理
  - **RateLimit拦截器**: 令牌桶限流
  - **Retry拦截器**: 自动重试、幂等性检查
  - **Caching拦截器**: 缓存命中/未命中、TTL、失效

- **系统观测性测试** (ST-I-001~005):
  - 端到端分布式追踪（Jaeger）
  - Prometheus指标导出
  - 结构化日志聚合
  - 熔断器激活和恢复
  - 拦截器链故障隔离

- **安全测试** (SEC-I-001~003):
  - 拦截器注入攻击防护
  - 追踪上下文注入防护（log4shell）
  - 认证拦截器绕过防护

### 3. **负载均衡和故障转移** (10 个测试)
- **负载均衡策略** (TC-LB-001~003):
  - RoundRobin轮询
  - Random随机
  - LatencyWeighted延迟加权

- **健康度评分和故障转移** (TC-LB-004~007):
  - 健康度评分计算
  - 实例过滤
  - 实例故障自动转移
  - 重试耗尽处理

- **高级特性** (TC-LB-008):
  - 单次调用的策略覆盖

### 4. **Eventgroup事件增强** (11 个测试)
- **事件组管理** (TC-EG-001~002):
  - 事件组注册
  - 订阅管理（SD协议）

- **事件流处理** (TC-EG-003~006):
  - 异步迭代器模式
  - 事件过滤
  - 事件历史/重传
  - 多客户端订阅

- **高级功能** (TC-EG-007~010):
  - 取消订阅和清理
  - 序列号追踪
  - 大型事件负载
  - 内部事件总线集成

### 5. **SOME/IP协议深度覆盖** (10 个测试)
- **协议基础** (PROTO-001~004):
  - SOME/IP报文头验证
  - 返回服务（RxSD）机制
  - 会话和序列号管理
  - 方法和事件多路复用

- **服务发现** (PROTO-005~006):
  - Eventgroup订阅（SD Subscribe）
  - 多客户端订阅

- **高级特性** (PROTO-007~010):
  - 端点地址动态更新
  - Eventgroup TTL和重订阅
  - 版本兼容性
  - 并发方法和事件

### 6. **安全性强化** (新增7个测试)
所有新的安全测试都集中在：
- 参数注入防护
- 反序列化安全
- 认证/授权绕过防护
- 拦截器安全性

### 7. **架构功能扩展** (新增4个扩展测试)
- **配置热重载**: 针对Method的timeout/retry动态更新
- **健康监控**: RPC特定的健康度和熔断器监控
- **拦截器链**: 执行顺序和短路行为验证
- **插件系统**: 自定义RPC编解码器支持
- **服务依赖**: Method级别的依赖注入
- **内部事件**: Method生命周期事件

### 8. **非功能性需求扩展** (新增3个扩展测试)
- **内存开销**: 每个Method的内存消耗（< 10KB）
- **RPC延迟**: 本地loopback p99 < 50ms，CPU < 2%
- **协议效率**: RPC开销 < 12字节，大负载效率 > 90%
- **启动性能**: 1000个Method在2秒内注册
- **故障转移**: 实例故障转移MTTR < 1秒
- **并发限制**: 每实例10000+并发RPC调用
